# ESP32 API Contract

This document defines the HTTP and WebSocket contract between the Smart Pill Reminder Box web/mobile app and the ESP32 firmware.

The system uses:

- **HTTP** for atomic configuration, recovery, and acknowledgement requests.
- **WebSocket** for live ESP32-generated feedback events.

The app does **not** poll the ESP32 for live reminder state. Runtime reminder changes are reported through WebSocket.

Base HTTP URL:

```text
http://<esp32-ip>
```

Base WebSocket URL:

```text
ws://<esp32-ip>/ws/events
```

All HTTP responses use:

```http
Content-Type: application/json
```

---

# Common Data Models

## Drawer

A drawer represents one physical compartment in the pill box.

```json
{
  "id": 1,
  "medicationName": "Aspirin",
  "enabled": true,
  "pillCount": 10
}
```

`pillCount` is the current number of pills remaining in the drawer. When a scheduled dose is completed, the ESP32 decrements this value by one. When it reaches `0`, the ESP32 emits a `DRAWER_EMPTY` event.

## Schedule

A schedule links a drawer to one or more reminder times.

A single schedule can contain multiple daily times because the same medication may need to be taken more than once per day.

```json
{
  "id": 1,
  "drawerId": 1,
  "times": [
    { "hour": 8, "minute": 0 },
    { "hour": 20, "minute": 30 }
  ],
  "daysOfWeek": ["MON", "TUE", "WED", "THU", "FRI"],
  "enabled": true
}
```

## Dose Event

A dose event is generated by the ESP32 when something relevant happens during the reminder lifecycle.

```json
{
  "id": 101,
  "type": "DOSE_COMPLETED",
  "scheduleId": 1,
  "scheduledTime": "08:00",
  "drawerId": 1,
  "medicationName": "Aspirin",
  "timestamp": "2026-06-13T08:02:00",
  "status": "TAKEN"
}
```

Important clarification: `TAKEN` means the correct drawer interaction was completed. The system does not prove that the medication was swallowed.

Known event types:

- `REMINDER_STARTED`
- `DRAWER_OPENED`
- `DRAWER_CLOSED`
- `DOSE_COMPLETED`
- `DOSE_MISSED`
- `DRAWER_EMPTY`

# 1. GET Requests

## 1.1 `getDeviceHealth`

Checks whether the ESP32 is reachable and ready.

### Request

```http
GET /api/health
```

### Request body

```json
{}
```

### Success response

```json
{
  "success": true,
  "deviceName": "Smart Pill Reminder Box",
  "firmwareVersion": "1.0.0",
  "wifiConnected": true,
  "webSocketEnabled": true,
  "storageReady": true
}
```

### Error response

```json
{
  "success": false,
  "error": "DEVICE_NOT_READY",
  "message": "Device services are not fully initialized"
}
```

---

## 1.2 `getDrawers`

Returns the drawer configuration stored on the ESP32.

### Request

```http
GET /api/drawers
```

### Request body

```json
{}
```

### Success response

```json
{
  "success": true,
  "drawers": [
    {
      "id": 1,
      "medicationName": "Aspirin",
      "enabled": true,
      "pillCount": 10
    },
    {
      "id": 2,
      "medicationName": "Vitamin D",
      "enabled": true,
      "pillCount": 30
    }
  ]
}
```

### Success response with no configured drawers

```json
{
  "success": true,
  "drawers": []
}
```

### Error response

```json
{
  "success": false,
  "error": "DRAWERS_UNAVAILABLE",
  "message": "Drawer configuration could not be loaded"
}
```

## 1.4 `getEvents`

Returns ESP32-generated events that have not been acknowledged by the app.

This request is mainly for recovery after a WebSocket disconnection, Wi-Fi interruption, or app restart. Normal live feedback should arrive through WebSocket.

### Request

```http
GET /api/events
```

### Request body

```json
{}
```

### Success response with events

```json
{
  "success": true,
  "events": [

      "id": 101,
      "type": "REMINDER_STARTED",
      "scheduleId": 1,
      "scheduledTime": "08:00",
      "drawerId": 1,
      "medicationName": "Aspirin",
      "timestamp": "2026-06-13T08:00:00"
    },
    {
      "id": 102,
      "type": "DOSE_COMPLETED",
      "scheduleId": 1,
      "scheduledTime": "08:00",
      "drawerId": 1,
      "medicationName": "Aspirin",
      "timestamp": "2026-06-13T08:02:00",
      "status": "TAKEN"
    },
    {
      "id": 103,
      "type": "DRAWER_EMPTY",
      "scheduleId": 1,
      "scheduledTime": "08:00",
      "drawerId": 1,
      "medicationName": "Aspirin",
      "timestamp": "2026-06-13T08:02:00",
      "status": "PENDING"
    }
  ]
}
```

### Success response with no events

```json
{
  "success": true,
  "events": []
}
```

### Error response

```json
{
  "success": false,
  "error": "EVENTS_UNAVAILABLE",
  "message": "Unacknowledged events could not be loaded"
```

# 2. PUT Requests

## 2.1 `updateDrawer`

Updates the medication and pill inventory assigned to a physical drawer.

Use this when the caregiver changes the medication name, enables/disables a drawer, or refills the drawer.

### Request

```http
PUT /api/drawers/{drawerId}
```

Example:

```http
PUT /api/drawers/1
```

### Request body

```json
{
  "medicationName": "Aspirin",
  "enabled": true,
  "pillCount": 10
}
```

`pillCount` is optional for backward compatibility. If omitted, the ESP32 keeps the current stored pill count.

### Success response

```json
{
  "success": true,
  "message": "Drawer updated",
  "drawer": {
    "id": 1,
    "medicationName": "Aspirin",
    "enabled": true,
    "pillCount": 10
  }
}
```

### Error response

```json
{
  "success": false,
  "error": "DRAWER_NOT_FOUND",
  "message": "Drawer 1 does not exist"
}
```

### Error response when pill count is invalid

```json
{
  "success": false,
  "error": "INVALID_DRAWER",
  "message": "Drawer pillCount cannot be negative"
}
```

---

## 2.2 `updateSchedule`

Updates an existing schedule by its schedule ID.

Use this when the app modifies the drawer assignment, reminder times, day list, or enabled state of an existing schedule.

### Request

```http
PUT /api/schedules/{scheduleId}
```

Example:

```http
PUT /api/schedules/2
```

### Request body

```json
{
  "drawerId": 1,
  "times": [
    { "hour": 8, "minute": 0 },
    { "hour": 20, "minute": 30 }
  ],
  "daysOfWeek": ["MON", "TUE", "WED", "THU", "FRI"],
  "enabled": true
}
```

### Success response

```json
{
  "success": true,
  "message": "Schedule updated",
  "schedule": {
    "id": 2,
    "drawerId": 1,
    "times": [
      { "hour": 8, "minute": 0 },
      { "hour": 20, "minute": 30 }
    ],
    "daysOfWeek": ["MON", "TUE", "WED", "THU", "FRI"],
    "enabled": true
  }
}
```

### Error response when schedule does not exist

```json
{
  "success": false,
  "error": "SCHEDULE_NOT_FOUND",
  "message": "Schedule 2 does not exist"
}
```

### Error response when drawer does not exist

```json

  "success": false,
  "error": "INVALID_DRAWER_ID",
  "message": "Schedule references drawer 9, but drawer 9 does not exist"
}
```

The app should use this when the caregiver adds a new reminder pattern for a drawer.

### Request


### Request body

```json
  "times": [
    { "hour": 8, "minute": 0 },
```

```json
  "success": true,

  "schedule": {
    "id": 3,
    "drawerId": 1,
    "times": [
      { "hour": 8, "minute": 0 },

    ],
```


---

## 3.2 `acknowledgeEvents`

Confirms that the app received and stored one or more ESP32-generated events.

After acknowledgement, the ESP32 can remove those events from its unacknowledged event buffer.

### Request

```http
POST /api/events/ack
```

### Request body

```json
{
  "eventIds": [101, 102]
}
```

### Success response

```json
{
  "success": true,
  "message": "Events acknowledged",
  "acknowledgedCount": 2
}
```

### Error response

```json
{
  "success": false,
  "error": "INVALID_EVENT_IDS",
  "message": "One or more event IDs do not exist in the unacknowledged event buffer"
}
```

# 5. PATCH Requests

No PATCH requests are defined for the MVP.

The API uses atomic `PUT`, `POST`, and `DELETE` operations instead.

# 6. WebSocket Event Contract

## 6.1 Connect to event socket

The app connects to this endpoint to receive live ESP32-generated feedback events.

### Connection

```text
ws://<esp32-ip>/ws/events
```

---

## 6.2 Event: `REMINDER_STARTED`

Sent when the ESP32 activates the buzzer, LEDs, and LCD for a scheduled dose.

### Event body

```json
{
  "id": 201,
  "type": "REMINDER_STARTED",
  "scheduleId": 1,
  "scheduledTime": "08:00",
  "drawerId": 1,
  "medicationName": "Aspirin",
  "timestamp": "2026-06-13T08:00:00"
}
```

---

## 6.3 Event: `DRAWER_OPENED`

Sent when the correct drawer is opened during an active reminder.

### Event body

```json
{
  "id": 202,
  "type": "DRAWER_OPENED",
  "scheduleId": 1,
  "scheduledTime": "08:00",
  "drawerId": 1,
  "medicationName": "Aspirin",
  "timestamp": "2026-06-13T08:01:00"
}
```

---

## 6.4 Event: `DRAWER_CLOSED`

Sent when the correct drawer is closed again after being opened.

### Event body

```json
{
  "id": 203,
  "type": "DRAWER_CLOSED",
  "scheduleId": 1,
  "scheduledTime": "08:00",
  "drawerId": 1,
  "medicationName": "Aspirin",
  "timestamp": "2026-06-13T08:02:00"
}
```

---

## 6.5 Event: `DOSE_COMPLETED`

Sent when the full drawer open/close interaction is completed.

### Event body

```json
{
  "id": 204,
  "type": "DOSE_COMPLETED",
  "scheduleId": 1,
  "scheduledTime": "08:00",
  "drawerId": 1,
  "medicationName": "Aspirin",
  "timestamp": "2026-06-13T08:02:00",
  "status": "TAKEN"
}
```

### Clarification

`TAKEN` means that the correct drawer interaction was completed. The system does not prove that the medication was swallowed.

---

## 6.6 Event: `DOSE_MISSED`

Sent when the reminder time window expires without completing the drawer interaction.

### Event body

```json
{
  "id": 205,
  "type": "DOSE_MISSED",
  "scheduleId": 1,
  "scheduledTime": "08:00",
  "drawerId": 1,
  "medicationName": "Aspirin",
  "timestamp": "2026-06-13T08:05:00",
  "status": "MISSED"
}
```

---

## 6.7 Event: `DRAWER_EMPTY`

Sent after a completed dose decrements the drawer inventory to `0`.

The caregiver app should treat this as an inventory alert and prompt for refill.

### Event body

```json
{
  "id": 206,
  "type": "DRAWER_EMPTY",
  "scheduleId": 1,
  "scheduledTime": "08:00",
  "drawerId": 1,
  "medicationName": "Aspirin",
  "timestamp": "2026-06-13T08:02:00",
  "status": "PENDING"
}
```
